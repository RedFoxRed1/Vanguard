🎮 Community Commands

🟢 Everyone

🟢 Everyone

📆 Events

/event create <title> <date> <time> [options]

🟢 Everyone
Creates an event, posts it as an embed with RSVP buttons (✅ Going / 🤔 Maybe / ❌ Can't Make It), and creates a native Discord scheduled event in the Events tab.

Options

Option Effect
location Venue or location string
description Event body text
post_channel Text or forum channel to post in (defaults to current)
notify_role Role to ping when posted
repeat None / Daily / Weekly auto-repost
duration Hours (default 2, range 0.5–48)
color Embed hex color (e.g. FF5733)
image_url Banner image URL

Tip

  • Posting to a forum channel creates a new thread.
  • Date/time is parsed in your personal timezone if you've configured one via /event settings.
  • The native scheduled event uses the post URL as its location when no location is given.

🎙️ Transcription

/transcribe start <output>

🟢 Everyone · must be in a voice channel
Joins your voice channel and posts live transcription to the chosen text channel. Each utterance appears as 🎙️ **Name**: text, and the command spins up a named session (e.g. silent-wolf). Use /leave to end it.

Note

Transcription engine priority

  1. Deepgram (cloud, fastest) — used when DEEPGRAM_API_KEY is set and Deepgram is healthy.
  2. Vosk + Whisper (local fallback) — Vosk shows a live preview, Whisper finalizes. Kicks in automatically if Deepgram times out or errors.
Command Perm Description
/transcribe sessions [page] 🟢 Lists past sessions (10/page): ID, voice channel, who started it, duration, and line count.
/transcribe view <session_id> 🟢 Shows the full transcript. Sends a .txt attachment if over 1900 characters.
/transcribe mode <engine> 🟢 Switches engine: deepgram re-enables cloud mode, local forces Vosk + Whisper. Persists across restarts.
/transcribe services <action> 🟣 Controls the background Vosk + Whisper processes (pre-loaded at bot startup): start spawns them if stopped, stop kills them to free RAM/VRAM (they respawn on the next /transcribe start), status shows whether Whisper is running and Vosk is ready or still loading.

🎙️ Voice AI

/join

🟢 Everyone · must be in a voice channel
Deploys Vanguard into your voice channel with AI voice interaction. It listens for the "Vanguard" wake word, transcribes via GPU Whisper, routes to TabbyAPI for a response, and speaks back through Kokoro ONNX TTS.

Spoken voice commands — require the wake word plus a configured code phrase:

Phrase Action
lock / unlock Locks / unlocks the voice channel
mute [name] / unmute [name] Server mutes / unmutes a member
leave Disconnects Vanguard
knowledge [key] to/is [value] Updates the AI knowledge base

/leave

🟢 Everyone
Disconnects Vanguard and stops the transcription session.

/vanguardvc enable <command> · disable <command> 🔴 Enables/disables individual voice commands.

Note

How AI search works
When someone asks a question in an AI-enabled channel, the bot runs web search (Firecrawl primary, Brave fallback), weather (if a location is detected), Wikipedia, and all matching pinned sources in parallel. Weather is dynamic (e.g. "weather in Dallas" → live from wttr.in), and Wikipedia auto-detects airsoft context, trying {topic} airsoft first to avoid pulling unrelated articles.

📅 Weekly Role & Attendance — /assignweeklyrole

The weekly role system tracks attendance by watching who types here in a designated channel during a rolling 7-day cycle.

✅ Eligible roles🚫 Excluded by default

Members need at least one (matched case-insensitively):

  • Recruit / Recruits
  • Operator / Operators
  • Senior Operator / Senior Operators
  • Board Member / Board Members / Board

Flags can include them when needed:

  • CEO, CFO, COO — command staff
  • Active Duty

Note

Check-in window

  • Start: midnight of the cycle start day — catches members who typed here before the scheduled trigger fired.
  • End: trigger time + 7 days (collector hard cutoff).
  • Messages outside this window are ignored by validation.

Scheduling

/assignweeklyrole schedule <role> <day> <time> <here_channel> <info_channel> <sheet>

🔴 Admin
Schedules a weekly role reset + check-in cycle and assigns a random codename (e.g. overlord).

Argument Meaning
role The weekly role to reset/assign
day Day of week (e.g. Monday)
time 24h format (HH:mm)
here_channel Where members type here
info_channel Where the announcement embed is posted
sheet Google Sheet tab to log attendance to

When the job fires it:

  1. Strips the role from all eligible members (skips admins)
  2. Posts the announcement embed
  3. Backfills any here messages sent since midnight (catches pre-trigger check-ins)
  4. Takes an initial snapshot of missing members
  5. Starts the 7-day collector
Command Perm Description
/assignweeklyrole trigger <job_id> [force] [no_announcement] [skip_attended] 🔴 Fires a job immediately. force runs even if it already ran today; no_announcement skips the embed; skip_attended keeps the role on members who already checked in today.
/assignweeklyrole list 🔴 Lists active jobs with codenames, roles, and schedules.
/assignweeklyrole clear 🔴 Cancels and removes all scheduled jobs.

Announcements

Command Perm Description
/assignweeklyrole set_announcement <job_id> [description] 🔴 Sets custom text for the announcement embed. Leave blank to reset to default.
/assignweeklyrole update_announcement <new_description> [job_id] 🔴 Edits the last posted announcement embed in place, without re-running the job.

Managing Attendance

/assignweeklyrole addrole <job_id> <member> [timestamp]

🔴 Admin
Manually grants the weekly role and logs the member for the current cycle. Use timestamp to backdate the check-in (2026-05-18 14:30 or ISO).

  • Skips if already logged this cycle (no duplicates)
  • Flagged as manual in the log and audit trail
  • Shows as a "manual entry" in validate since the member never typed here

/assignweeklyrole attendance [job_id] [by_day] [all] [export]

🔴 Admin
Views attendance records. Default shows the current cycle for the auto-detected (or specified) job.

  • by_day: True — breaks check-ins down by calendar day
  • all: True — shows all historical cycles (up to 10 embeds)
  • export: True — downloads a CSV instead. Combined with all: True, sends only the file plus a summary line (avoids Discord's 6000-char embed limit)

CSV columns: Cycle Week · Job ID · Display Name · Username · User ID · Check-in Time (UTC) · Synced · Manual

/assignweeklyrole sync [job_id]

🔴 Admin
Pushes unsynced local attendance entries to Google Sheets. Leave job_id blank to sync all jobs. Reports pushed vs. failed counts.

/assignweeklyrole validate [job_id]

🔴 Admin
Runs a validation pass on the active cycle (also runs automatically every 4 hours while a cycle is active). It checks for:

Check What it does
Duplicates Removes duplicate log entries, keeping the earliest
Role restored Re-adds the role to anyone in the log who lost it
Missing from log Adds anyone who typed here but has no log entry
Manual entries Flags members who never typed here — informational, not errors
Unsynced Lists entries not yet pushed to Google Sheets

/assignweeklyrole missing [job_id] [include_command_staff] [include_active_duty] [export]

🔴 Admin
Shows eligible members who haven't checked in this cycle, sorted by most total cycles missed.

Default behavior excludes CEO/CFO/COO and Active Duty, and only counts members with an eligible role (Recruit, Operator, Senior Operator, Board Member).

Flags: include_command_staff: True, include_active_duty: True, and export: True (sends a CSV instead of the embed).

Tip

  • Missing count includes the current cycle (so a first miss = 1).
  • Tracks both total and consecutive missed cycles — a check-in resets the consecutive streak but not the total.
  • Members at 26+ consecutive missed cycles (~6 months) are flagged 🚨 — the inactivity threshold from the promotion requirements that subjects a member to demotion/removal review.
  • Repeat offenders (2+ cycles missed) are tallied separately in the summary.
  • Running this saves a snapshot used to track late check-ins through the week.

CSV columns: Display Name · Username · User ID · Cycles Missed · Consecutive Missed · At 6-Month Threshold (sorted by most missed, descending)

/assignweeklyrole set_alert_channel [job_id] [channel]

🔴 Admin
Sets the channel for automatic inactivity alerts. When a cycle ends and a member reaches 26 consecutive missed cycles (~6 months), the bot posts a red alert embed naming them — once per member; the flag resets if they check in again. Leave channel blank to disable alerts for the job.

🤖 Passive Behavior

Things Vanguard does automatically — no command required.

Feature What happens
🧠 AI Chat In configured channels, the bot replies automatically. Text answers come from Claude Haiku (cloud) when an Anthropic API key is set, falling back to the local model on any failure; voice always uses the local model. Context includes web search (Firecrawl primary, Brave fallback) with scraped page text from the top 3 results, dynamic weather (if a location is detected), a Wikipedia lookup (airsoft context first when relevant), and any pinned sources matching the query. Replies can also include images: relevant image URLs found in the scraped results (restricted to URLs actually present in the search context), and local library images auto-attached when their keyword matches (/vanguard imgadd). Managed via /vanguard.
🚫 Word Filter When enabled, every message is checked against the filter list and auto-deleted on a match. Violations log to the configured channel.
🎂 Birthday Announcements On startup and on a daily check, posts a birthday message when a member's birthday is set and an announcement channel is configured.
⏲️ Temp-Ban Expiry Temp-bans are tracked in tempbans.json and automatically reversed when their timer expires.
📝 Message Logging With a log channel set, deleted and edited messages are logged with before/after content.
👋 Member Join/Leave Logging With a member-log channel set, join and leave events post with the member's name and ID.
🎉 Welcome Messages With a welcome channel + message set, a welcome is sent on join. {user} is replaced with the member's mention.
🎖️ Rank Tenure Tracking Whenever a rank role changes on any member (Recruit, Operator, Senior Operator, Board, CEO/CFO/COO), the gain date is recorded automatically. Losing a rank clears its record, so a demotion restarts the tenure clock. On startup the bot also sweeps the audit log (last ~45 days, needs the View Audit Log permission) to fill gaps from promotions made while it was offline — never overwriting existing records. Shown in /userinfo; backfill ranks older than ~45 days with /settenure.
📋 Tenure Milestones With a channel set via /botconfig set_tenure_channel, the bot checks tenure at startup and every 12 hours and announces members who crossed a promotion-eligibility timeframe: 3 months as Recruit (probation complete), 6 months as Operator (Sr. Operator nomination), 6 months as Senior Operator (elected-role eligibility). The first-ever check records existing eligibility silently (no catch-up wall) — only new crossings announce, once per member+milestone. See the full current roster anytime with /eligible. Tenure eligibility only — other requirements still apply.
🚨 Inactivity Alerts With an alert channel set via /assignweeklyrole set_alert_channel, the bot checks consecutive missed check-ins at the end of every weekly cycle and posts an alert when a member crosses 26 consecutive misses (~6 months) — the inactivity threshold for demotion/removal review. One alert per member; checking in resets the flag.
